## Основные характеристики каждого окна:
### Окно Блэкмана-Харриса:
#### Преимущества и недостатки
Окно Блэкмана-Харриса (Blackman-Harris window) - это один из наиболее популярных оконных функций в цифровой обработке сигналов, особенно в спектральном анализе. Вот его основные преимущества и недостатки:

Преимущества:
1. Высокое подавление боковых лепестков
   Обеспечивает очень низкий уровень боковых лепестков (около -92 дБ для минимальной 4-членной версии)

Это одно из лучших окон по подавлению утечки спектра (spectral leakage)

2. Хорошая частотная избирательность
   По сравнению с другими окнами с аналогичным подавлением боковых лепестков имеет оптимальный компромисс между шириной главного лепестка и уровнем боковых

3. Несколько версий с разными компромиссами
   Существуют 3-членные, 4-членные и более сложные версии

4-членная версия (которая используется в вашем коде) обеспечивает максимальное подавление боковых лепестков

4. Аналитическая форма
   Простая математическая формула на основе косинусов

Легко вычисляется и оптимизируется

5. Широкое применение в точных измерениях
   Используется в высокоточных спектроанализаторах

Применяется в радиоастрономии, радарах, телекоммуникациях

Недостатки:
1. Широкий главный лепесток
   Главный лепесток примерно в 2 раза шире, чем у прямоугольного окна

Это снижает разрешающую способность по частоте

2. Потеря энергии сигнала
   Сильно затухает к краям, что приводит к потере информации

Не подходит для анализа коротких сигналов с важными краевыми эффектами

3. Высокая вычислительная сложность
   По сравнению с простыми окнами (Ханна, Хэмминга) требует вычисления трех косинусов

Хотя в современных системах это не критично

4. Неоптимально для некоторых задач
   Для задач, где важнее разрешение, а не подавление боковых лепестков, лучше подходят другие окна

5. Параметры фиксированы
   Коэффициенты жестко зафиксированы в стандарте

Меньше гибкости по сравнению с параметрическими окнами (как окно Кайзера)

#### Советы по окну Блэкмана-Харриса:
✅ Используйте, когда:
- Критически важно подавление спектральной утечки
- Нужно обнаружить слабые сигналы рядом с сильными
- Работаете с шумными данными
- Выполняете высокоточные спектральные измерения

❌ Не используйте, когда:
- Важно частотное разрешение
- Сигнал очень короткий
- Нужно сохранить временную локализацию
- Выполняете реальную обработку с ограниченными ресурсами

### Окно Хэмминга (Hamming):
Коэффициенты: α = 0.54, β = 0.46
Хороший компромисс между шириной главного лепестка и уровнем боковых лепестков
Боковые лепестки примерно -42 дБ

### Окно Хэннинга (Hann):
Также известно как "косинусное окно" или "raised cosine"
Коэффициенты: α = 0.5
Боковые лепестки убывают быстрее, чем у Хэмминга (-31 дБ первый боковой лепесток)

### Окно Тьюки (Tukey):
Параметрическое окно с плавными переходами
alpha = 0: прямоугольное окно (резкие края)
alpha = 1: окно Хэннинга (полностью плавные края)
Позволяет контролировать компромисс между шириной главного лепестка и утечкой спектра
